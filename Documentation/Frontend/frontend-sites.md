# Frontend Seiten

## Home Page
**Ziel**: Erster Einstieg und Einführung

**Beschreibung**:
Die Startseite ist der erste Berührungspunkt für Nutzer. Sie erklärt kurz, was die Plattform macht, und motiviert zur Registrierung.

**Inhalt**:
- Kurze Erklärung des Produktes
- Call-To-Action
- Features
    - Baukasten-Editor
    - Einfache Bedienung
    - Individuelles Design
- Navigation
    - Login / Registrierung

## Login / Registration Page
**Ziel**:
Benutzer authentifizieren

**Beschreibung**:
Seite für Login bestehender Nutzer und Registrierung neuer Accounts.

**Inhalt**:
- Login-Formular:
    - Email / Benutzername
    - Passwort
- Registrierung:
    - Email
    - Passwort
    - Passwort bestätigen
- Validierung & Fehlermeldungen

## Dashboard / Startseite (nach Login)
**Ziel**: Zentrale Übersicht für Nutzer

**Beschreibung**:
Nach dem Login landet der Nutzer hier und sieht alle seine Projekte.

**Inhalt**:
- Übersicht über alle Portfolios
- Button "Neues Portfolio erstellen"
- Portfolio aktionen:
    - Bearbeiten
    - Löschen
    - Veröffentlichen
- Letzte Änderungen / Versionen
- Account Einstellungen 

## Portfolio Editor (Kernelement)
**Ziel**: Hauptfunktion *Portfolio erstellen*

**Beschreibung**:
Der wichtigste Teil der App: ein Baukasten-Editor

**Inhalt**:
- Komponenten hinzufügen:
    - Text
    - Bilder
    - Projekte 
    - Skills
- Layout bearbeiten (Sections / Blöcke)
- Inhalte direkt editieren
- Speichern / Auto-Save
- Rückgängig machen
- Drag and Drop System

## Design / Theme Seite
**Ziel**: Individuelles Aussehen festlegen

**Beschreibung**:
Hier kann der Nutzer das Design seines Portfolios anpassen.

**Inhalt**:
- Auswahl von Themes (Templates)
- Anpassung:
    - Farben
    - Schriftarten

## Vorschau Seite (optional)
**Ziel**: Portfolio vor Veröffentlichung testen

**Beschreibung**:
Zeigt eine realistische Ansicht des Portfolios, wie es später öffentlich aussieht.

**Inhalt**:
- Vollständige Darstellung
- Responsives Verhalten testen
- Button
    - Veröffentlichen
    - Zurück zum Editor

## Öffentliche Portfolio Website
**Ziel**: Präsentation fertige Seite

**Beschreibung**:
Das veröffentlichte Portfolio ist über eine öffentliche URL erreichbar.

**Inhalt**:
- Kein Login erforderlich
- Individuelle URL (Verschlüsselung)
- Profil / Projekte / Skills
- Schnelle Ladezeiten
